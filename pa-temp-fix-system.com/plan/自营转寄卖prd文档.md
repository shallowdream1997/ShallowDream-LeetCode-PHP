# 一. 需求基本概况
## 1.1 业务背景
+ **<font style="color:rgb(15, 17, 21);">库存压力</font>**<font style="color:rgb(15, 17, 21);">：往年存在一批非S级新品（约1938W货值），因初期无寄卖商承接，由自营兜底承担货权。随着业务战略调整，需逐步降低自营在此类寄卖定位SKU上的库存压力。</font>
+ **<font style="color:rgb(15, 17, 21);">市场验证</font>**<font style="color:rgb(15, 17, 21);">：这批SKU经过市场验证，动销表现稳定，直接转给寄卖商能快速提升其业绩。</font>
+ **<font style="color:rgb(15, 17, 21);">盲目转换风险</font>**<font style="color:rgb(15, 17, 21);">：并非所有SKU都适合转寄卖，需建立数据化准入门槛，避免销售效率下降</font>

## 1.2 业务目的
+ **<font style="color:rgb(15, 17, 21);">明确映射关系</font>**<font style="color:rgb(15, 17, 21);">：明确当前货源工厂与寄卖商的映射关系。</font>
+ **<font style="color:rgb(15, 17, 21);">量化SKU池</font>**<font style="color:rgb(15, 17, 21);">：明确自营转寄卖可操作的SKU总池子数量。</font>
+ **<font style="color:rgb(15, 17, 21);">规范操作流程</font>**<font style="color:rgb(15, 17, 21);">：明确首批自营转寄卖的操作流程（不增加寄卖系数，以寄卖商竞标为前提）。</font>

## 1.3 业务价值
     核心指标：SCMS

+ **<font style="color:rgb(15, 17, 21);">保持销售效率</font>**<font style="color:rgb(15, 17, 21);">：终端售价不变，不增加寄卖系数，Margin不降。</font>
+ **<font style="color:rgb(15, 17, 21);">降低自营资金</font>**<font style="color:rgb(15, 17, 21);">：盘活1938W自营存量库存，缓解资金与周转压力。</font>
+ **<font style="color:rgb(15, 17, 21);">货源同源度</font>**<font style="color:rgb(15, 17, 21);">：引导寄卖商聚焦少数核心货源，形成规模效应。</font>
+ **<font style="color:rgb(15, 17, 21);">风险可控</font>**<font style="color:rgb(15, 17, 21);">：建立标准化的自营与寄卖货权互转机制。</font>

**<font style="color:rgb(15, 17, 21);">本需求是业务域的整体架构图的哪一部分</font>**<font style="color:rgb(15, 17, 21);">：供应链优化 - 库存管理 - 自营转寄卖流程模块。</font>

## 1.4 业务会议纪要（非必须）
会议纪要说明：

+ **<font style="color:rgb(15, 17, 21);">决策点</font>**<font style="color:rgb(15, 17, 21);">：不以增加寄卖系数的方式转接，采用“热销招募”形式发布。</font>
+ **<font style="color:rgb(15, 17, 21);">核心原则</font>**<font style="color:rgb(15, 17, 21);">：不给直接下调Cost的空间，让寄卖商自行与工厂谈判，发挥其谈判价值。</font>
+ **<font style="color:rgb(15, 17, 21);">激励导向</font>**<font style="color:rgb(15, 17, 21);">：谈判能力强则多接，提供充分的市场竞争空间。</font>

## 1.5 名词解释 （非必须）
1. **<font style="color:rgb(15, 17, 21);">A级寄卖商</font>**<font style="color:rgb(15, 17, 21);">：由寄卖组定义及提供的核心优质寄卖商名单。</font>
2. **<font style="color:rgb(15, 17, 21);">货源同源度</font>**<font style="color:rgb(15, 17, 21);">：指多个SKU来源于同一生产工厂或同一货源渠道的关联度。</font>
3. **<font style="color:rgb(15, 17, 21);">母鸡权</font>**<font style="color:rgb(15, 17, 21);">：指对该货源下所有SKU的供货权，通过评选获得。</font>
4. **<font style="color:rgb(15, 17, 21);">CE单</font>**<font style="color:rgb(15, 17, 21);">：寄卖商申请样品的订单单据，用于验证产品与市场匹配度。</font>
5. **<font style="color:rgb(15, 17, 21);">SKU覆盖率</font>**<font style="color:rgb(15, 17, 21);">：寄卖商已入库SKU数量 / 该清单总SKU数量。</font>

# 二. 产品方案概述
## 2.1 产品需求描述-页面
<font style="color:rgb(15, 17, 21);">搭建一套自营库存转寄卖的系统流程。系统根据“货源同源度”、历史销量等条件自动筛选符合转换条件的SKU并组建清单。面向寄卖商进行定时、竞标式招募。通过“开CE单-来货-入库”的流程，结合“SKU覆盖率”及“同源扶持”机制评选出最终的“母鸡权”持有者。同时，建立价格监控与补货监控机制，对履约不力的寄卖商进行清单回收。</font>

### <font style="color:rgb(15, 17, 21);">2.1.1 清单逻辑</font>
#### 2.1.1.1 清单脚本
+ 脚本说明
    - <font style="color:#DF2A3F;">*注意点所有参与组单skuid为自营skuid</font>
        * <font style="color:#DF2A3F;">若导入的skuid为寄卖，导入时报错，skuid存在业务类型为寄卖的skuid</font>
        * <font style="color:#DF2A3F;">组单逻辑脚本过滤掉寄卖skuid，初始来货标签必须为“寄卖转自营”且为APO状态，否则报错：SKU非“寄卖转自营”或状态非APO</font>
        * <font style="color:#DF2A3F;">货源工厂名称，必填。系统校验逻辑：必须与系统绑定的该SKU货源工厂一致，否则报错：SKU与系统绑定货源工厂不一致</font>
    - 脚本自动运行，部分脚本存在审核后执行，根据业务调研，可去除这部分审核，根据逻辑设置无需审核，系统自动审核通过（组单后的清单）
    - 脚本清单（组单、发布、分配(母鸡权）、自营转寄卖）
        * 组单脚本（*本次数据来源为：人为导入，根据导入的skuid遵循以下条件进行组单）
            + 同一货源，同一产品线组单
            + 每张清单sku数需在，10<sku数≤200，上限200个SKU，超出自动拆单，SKU不足10个或未被纳入清单的，继续保留在招募池中，等待下一轮
            + 组单按SKU创建时间顺序进行组单（优先组生成时间早的skuid）<font style="color:#DF2A3F;">按照a号的生成时间</font>
            + <font style="color:#DF2A3F;">组单组当天去查询近21天曾提醒过补货的SKU</font>
            + 近90天销量≥1
        * 发布脚本（发布审核通过的清单，每周三发布（当天00：00：01可见），14点可抢）
        * 清单分配（每天21点执行查询覆盖率，进行分配）进行自营转寄卖，授予母鸡权）
+ **定时脚本机制**
    - 每天执行组单，根据条件进行扫描，组单完成，系统自动审核通过，等待发布脚本执行。~~**需开发经理**~~~~进行审核（根据条件规则执行清单进行组单）。~~
    - 每周三进行发布，<font style="color:rgb(64, 64, 64);"> 待组单完成后，脚本执行组单完成状态的清单进行清单发布，上限20张单</font>
    - 每天21点执行，查询CE单覆盖率，覆盖率高的分配（母鸡权）
+ **<font style="color:rgb(64, 64, 64);">脚本执行日志：</font>**
    - <font style="color:rgb(64, 64, 64);">记录每个清单脚本执行时间、执行动作、操作结果（成功/失败）。</font>

    - 新设状态，每个执行脚本以状态枚举值进行控制
