# 4-2 招募清单详情

## 一、概述

| 项目 | 说明 |
|------|------|
| **PRD章节** | 2.1.3.2 招募清单详情 |
| **面向用户** | 运营后台管理人员 |
| **功能** | 查看某张清单的完整信息，包括基本资料、发布详情、供应商列表 |

---

## 二、数据源

### 2.1 基本资料

| 字段 | 来源 | 说明 |
|------|------|------|
| 清单类型 | 常量 | 固定"自营转寄卖" |
| 产品分类 | 关联分类表 | 根据 `category_full_path_id` |
| 产品线 | 关联分类表 | 末级中文分类 |
| 清单SKU数 | `recruit_list.sku_count` | 直接读取 |
| 平均销售预估 | `recruit_list.estimated_month_sale_qty` | 组单时快照 |
| 预估月销售总额 | `recruit_list.estimated_month_sale_amount` | 组单时快照 |
| 清单文件 | `recruit_list.file_url` | OSS文件链接，可下载 |
| 清单创建时间 | `recruit_list.group_time` | 组单完成时间 |

### 2.2 发布详情

| 字段 | 来源 | 说明 |
|------|------|------|
| 发布时间 | `recruit_publish.publish_begin_time` | 对应publish记录 |
| 发布操作人 | `recruit_list.publish_by` | 系统/用户名 |
| 参与寄卖商数 | `recruit_apply` | 动态统计：`COUNT(DISTINCT supplier_id)` |
| 分配寄卖商 | `recruit_list.award_supplier_id` | 分配后写入 |
| 分配时间 | `recruit_list.award_time` | 分配后写入 |
| 分配操作人 | `recruit_list.award_by` | 分配后写入 |

### 2.3 供应商列表

| 字段 | 来源 | 计算逻辑 |
|------|------|----------|
| 供应商名称 | `recruit_apply.supplier_name` | 直接读取 |
| 已有同货源SKU数 | 关联供应商同货源SKU表 | **动态查询** |
| 加入招募车时间 | `recruit_apply.create_time` | 创建时间 |
| CE开单时间 | `recruit_apply.ce_create_time` | CE系统回写 |
| CE单号 | `recruit_apply.ce_bill_no` | CE系统回写 |
| CE单状态 | 时间戳推断 | 详见CE子计划 |
| 入库覆盖率 | `recruit_apply.base_coverage_rate` | QC通过后更新 |
| 同源加权率 | `recruit_apply.same_source_weight_rate` | 组单时计算 |
| SKU覆盖率 | `recruit_apply.final_coverage_rate` | 以上两者之和 |
| 招募状态 | `recruit_apply.apply_status` | 枚举转换 |
| 招募结果 | `recruit_apply.award_result` | 评选后写入：成功/失败 |

## 三、查询流程

```
请求详情 recruitId
    │
    ├─ 1. 查询基本信息 ─────────────────────────────────────
    │    recruit_list WHERE id = recruitId
    │    关联分类表获取产品分类名称
    │
    ├─ 2. 查询发布详情 ─────────────────────────────────────
    │    recruit_publish WHERE recruit_id = recruitId
    │    ORDER BY publish_round DESC LIMIT 1（取最新一轮）
    │
    ├─ 3. 查询供应商列表 ──────────────────────────────────
    │    recruit_apply WHERE recruit_id = recruitId
    │    AND is_deleted = 0
    │    ORDER BY create_time ASC
    │
    └─ 4. 补充字段计算 ────────────────────────────────────
       每个供应商的CE状态（时间戳推断）
       已有同货源SKU数（关联查询）
```

---

## 四、状态走向

供应商列表中每个供应商的招募状态映射：

| 字段值 | `apply_status` | 说明 |
|--------|:--------------:|------|
| 已加入 | 10 | 已加入招募车，未开CE |
| 已开CE | 20 | 已开CE单 |
| 等待评选 | 30 | CE流程进行中 |
| 分配完成 | 40 | 已获胜分配 |
| 清单完成 | 50 | 清单结束 |
| 超时清出 | 90 | 超时未开单 |
| 放弃/作废 | 100 | 主动放弃或作废 |

## 五、表数据处理

| 操作 | 表 | 说明 |
|------|-----|------|
| SELECT | `recruit_list` | 主表信息 |
| SELECT | `recruit_publish` | 发布记录（最新一轮） |
| SELECT | `recruit_apply` | 供应商列表 |
| SELECT（关联） | `list_sku`/供应商SKU表 | 已有同货源SKU数 |

---

## 六、难点与解决点

| 难点 | 解决 |
|------|------|
| **CE单状态需要从时间戳推断** | 增加枚举 `CeStatusCalculator` 工具类，根据 `ce_bill_no/ce_create_time/ce_send_time/first_qc_pass_time` 时间戳范围推断状态 |
| **已有同货源SKU数**是跨模块查询 | 调用供应商模块的Feign接口：`supplierSkuApi.countByFactoryAndGroup(groupId, factoryId)` |
| **详情页加载性能** | 详情页数据量不大（1条list + N条apply），无需特殊优化，单SQL查询即可 |
| **文件下载链接时效** | OSS链接使用临时签名URL，有效期30分钟 |
