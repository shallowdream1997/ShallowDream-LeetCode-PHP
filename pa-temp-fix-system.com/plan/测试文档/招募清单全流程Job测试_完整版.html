二、Job 详细说明 6 Jobs

Job 1. 自动发布 AutoPublishJob

Handler: consignmentRecruitAutoPublishJobHandler Service: ConsignmentRecruitAutoPublishService 调度: 每周三 10:00 API: POST /v1/triggerAutoPublish

核心逻辑：从 recruit_list 读取 list_status=10(待发布) 的清单，发布为 20(招募中)，并创建 recruit_publish 发布轮次记录。

sku_status=20→30"] UPDATE_SKU --> LOG["INSERT action_log"] LOG --> LOOP style START fill:#e3f2fd,stroke:#1565c0 style CHECK fill:#f3e5f5,stroke:#7b1fa2 style INSERT fill:#fff3e0,stroke:#e65100 style UPDATE_LIST fill:#e8f5e9,stroke:#2e7d32
实体变更

Job 2. 评选开始 EvalStartJob

Job 3. CE到货/质检查询 CeArrivalCheckJob

Handler: consignmentRecruitCeArrivalCheckJobHandler Service: ConsignmentRecruitCeArrivalCheckService 调度: 每天 12:00 / 18:00 API: POST /v1/triggerCeArrivalCheck

核心逻辑：仅查 status=30(等待评选) 的 apply，通过CE系统 surfaceOn 字段判断质检完成，计算覆盖率+同源加权。

graph TD TRIGGER["12:00/18:00触发"] --> S1["Step 1: 查询待检查申请
apply_status=30 AND ce_bill_no非空"] S1 --> LOOP{"逐条检查"} LOOP -->|"有数据"| CE["2a. 解析 ce_bill_no
逗号分隔多个CE单号"] CE --> SKU["2b. 获取清单SKU列表"] SKU --> FEIGN["2c. 批量查询CE明细
CetsBillQueryService.getBillDetailByBillNo
M次Feign调用"] FEIGN --> QC{"surfaceOn非空?
(质检完成)"} QC -->|"否"| SKIP["跳过,等下次轮询"] QC -->|"是"| FIRST{"首次质检通过?"} FIRST -->|"是"| SETTIME["2e. 设 firstQcPassTime"] SETTIME --> COVERAGE["2f. 触发覆盖率计算
CoverageCalcService.handleQcPassEvent"] FIRST -->|"否"| COVERAGE COVERAGE --> LOG["2g. INSERT action_log"] LOG --> LOOP QC -->|"否"| LOOP LOOP -->|"无数据"| END["正常结束"] style TRIGGER fill:#e3f2fd,stroke:#1565c0 style FEIGN fill:#fff3e0,stroke:#e65100 style QC fill:#f3e5f5,stroke:#7b1fa2 style COVERAGE fill:#e8f5e9,stroke:#2e7d32
覆盖率计算： inboundSkuCount = 累计入仓SKU数（增量累加）
skuCoverageRate = inboundSkuCount * 100 / skuCount（百分比制）
sameSourceWeight = sameSourceSkuCount > 0 ? 10 : 0（同源SKU数>0时+10%）
finalCoverageRate = skuCoverageRate + sameSourceWeight
实体变更
recruit_applyinbound_sku_count, final_coverage_rate 增量更新
recruit_applyfirst_qc_pass_time 首次质检完成时设置

四、测试验证SQL速查 SQL

-- 1. 检查清单状态
SELECT id, recruit_no, list_status, award_time, publish_end_time
FROM scms_consignment_recruit_list WHERE id = ?;

-- 2. 检查招募车状态
SELECT id, recruit_id, supplier_id, apply_status, first_qc_pass_time,
       inbound_sku_count, final_coverage_rate, award_result
FROM scms_consignment_recruit_apply WHERE recruit_id = ?;

-- 3. 检查发布轮次
SELECT id, recruit_id, publish_round, publish_status, publish_end_time
FROM scms_consignment_recruit_publish WHERE recruit_id = ?;

-- 4. 检查SKU状态
SELECT id, recruit_id, sku_id, sku_status
FROM scms_consignment_recruit_list_sku WHERE recruit_id = ?;

-- 5. 检查操作日志
SELECT id, recruit_id, apply_id, action, operator_name, content, create_time
FROM scms_consignment_action_log WHERE recruit_id = ? ORDER BY create_time;
