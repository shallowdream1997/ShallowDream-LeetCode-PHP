1. 招募清单列表分页（增强）POST

寄卖商查看招募清单列表，含丰富筛选、竞争池、同源母鸡数、统计栏，支持展开SKU行。

① 筛选栏：发布时间、工厂、分类、寄卖总价范围、SKU ID（textarea）、同源开关、竞争池人数
② 统计栏：共 N 条货源，共 N 个SKU，我的招募车 N 个

2. 加入招募车 POST

POST/scms/consignment/recruit/supplier/v1/cart/join
原型操作：清单列表页操作列 → 点击「加入招募车」→ 确认弹窗 → 调用本接口。
前置条件：清单状态=招募中(20)，竞争池未满。

请求参数

字段类型必填说明标注
recruitNoString是招募清单编号入参
supplierIdLong是寄卖商ID入参
groupIdLong是寄卖商集团ID入参

响应

字段类型说明
valueBoolean是否加入成功

业务逻辑

  1. 根据 recruitNo 查找 recruit_list
  2. 申请Redis锁 consignment:recruit:lock:{recruitId}
  3. 检查是否已加入（防重复）
  4. 乐观锁更新竞争池 +1（WHERE supplier_apply_count < 5）
  5. 新增申请记录 apply_status=10
  6. 释放Redis锁

请求示例

{"recruitNo":"SC20260520001","supplierId":1001,"groupId":2001}

响应示例

{"code":200,"message":"success","data":{"value":true}}

二、公共枚举说明 枚举

2.1 招募清单状态（listStatus）

编码名称说明
10待发布等待定时任务自动发布或后台手动发布
20招募中开放招募，竞争池未满
25已抢完竞争池已满5人
30待审核招募结束，等待审核
40审核通过审核通过
50评选中正在执行评选
60评选完成评选已完成
70已分配已分配供应商
90已过期招募已过期
100已作废后台作废

2.2 申请状态（applyStatus）

编码名称说明
10已加入寄卖商已加入招募车
20已开CE已创建CE单
30等待评选已进入评选池
40分配完成评选获胜，分配完成
90超时清出超时未开CE被清出
100放弃/作废主动撤回、手动移除或作废
 
stateDiagram-v2
    [*] --> 已加入: joinCart
    已加入 --> 已开CE: createCe/appendSkuToCe
    已加入 --> [*]: withdrawCart(物理删除)
    已开CE --> 等待评选
    等待评选 --> 分配完成: 获胜
    等待评选 --> 放弃: 落选
    已加入 --> 超时清出
    已加入 --> 放弃

2.3 评选结果（awardResult）

编码名称说明
0未评选尚未执行评选
1获胜评选获胜
2落选评选落选

2.4 同货源标识（sameSourceFlag）

编码名称说明
0非同源无合作母鸡SKU
1同源已有合作母鸡SKU

2.5 CE状态（ceStatus 推断值）

值推断条件说明
未开单ceBillNo IS NULL尚未创建CE单
已开单ceBillNo IS NOT NULL已创建CE单号
已发货ceSendTime IS NOT NULLCE已发货
处理完成firstQcPassTime IS NOT NULL首次质检已通过
