## 三、24个配置项说明表

### 3.1 招募发布参数

| 配置项 | 类型 | 默认值 | 说明 | 对应硬编码位置 |
|---|---|---|---|---|
| `publishDayOfWeek` | int | 3 | 发布日（1=周一，3=周三） | 计划文档定义 |
| `publishVisibleTime` | string | "10:00:00" | 清单可见时间 | AutoPublishService.calcPublishTimeNodes() |
| `applyStartTime` | string | "14:00:00" | 开放申请时间（可抢单） | AutoPublishService.calcPublishTimeNodes() |
| `applyEndTime` | string | "21:00:00" | 申请结束时间 | AutoPublishService.calcPublishTimeNodes() |
| `publishValidHours` | int | 24 | 发布报名链接有效期（小时） | 原ConsignmentRecruitConfigBO已有字段 |
| `maxPublishRoundPerMonth` | int | 4 | 每月最大发布轮次 | 原ConsignmentRecruitConfigBO已有字段 |
| `roundMaxPublishCount` | int | 20 | 每轮最多发布的清单数 | 计划文档定义 |

### 3.2 清单与SKU参数

| 配置项 | 类型 | 默认值 | 说明 | 对应硬编码位置 |
|---|---|---|---|---|
| `listMaxSkuCount` | int | 200 | 单张清单最大SKU数，超出自动拆单 | 计划文档定义 |
| `listMinSkuCount` | int | 11 | 单张清单最小SKU数（必须>10） | 计划文档定义 |
| `listMaxApplyCount` | int | 5 | 每张清单最多允许的寄卖商申请数（竞争池容量） | 计划文档定义 |
| `supplierMaxRecruitSlot` | int | 5 | 每个寄卖商最多可加入的招募清单数 | 计划文档定义 |

### 3.3 评选参数

| 配置项 | 类型 | 默认值 | 说明 | 对应硬编码位置 |
|---|---|---|---|---|
| `directAwardCoverageRate` | decimal | 0.80 | 覆盖率>=此值直接获胜（小数，0.80=80%） | AutoAwardService.DIRECT_AWARD_THRESHOLD |
| `awardTimeoutDays` | int | 7 | 评选超时时间（天） | 原ConsignmentRecruitConfigBO已有字段 |
| `awardWaitDays` | int | 14 | 首个CE质检通过后的最长等待天数 | AutoAwardService.AWARD_WAIT_DAYS |
| `sameSourceWeightRate` | decimal | 0.10 | 同源扶持加权比例（小数，0.10=10%） | CoverageCalcService.SAME_SOURCE_WEIGHT |

### 3.4 超时与回收参数

| 配置项 | 类型 | 默认值 | 说明 | 对应硬编码位置 |
|---|---|---|---|---|
| `ceCreateTimeoutDays` | int | 3 | 加入招募车后开CE单超时天数（自然日） | CeTimeoutCleanService.CE_TIMEOUT_DAYS |
| `recycleCoverageThreshold` | decimal | 0.80 | 回收覆盖率阈值（小数，0.80=80%） | RecycleCheckService.RECYCLE_COVERAGE_THRESHOLD |
| `recycleConsecutiveMonthsWarn` | int | 2 | 连续N月低于阈值触发预警 | RecycleCheckService.WARN_CONSECUTIVE_MONTHS |
| `recycleConsecutiveMonthsRecycle` | int | 3 | 连续N月低于阈值回收清单 | RecycleCheckService.RECYCLE_CONSECUTIVE_MONTHS |

### 3.5 组单参数

| 配置项 | 类型 | 默认值 | 说明 | 对应硬编码位置 |
|---|---|---|---|---|
| `groupSkuOrderField` | string | "create_time" | 组单排序字段（按SKU创建时间） | 计划文档定义 |
| `groupSkuOrderAsc` | bool | true | 组单排序方向（true=正序） | 计划文档定义 |
| `saleDaysForGroup` | int | 90 | 组单筛选的近N天销量条件 | 计划文档定义 |
| `minSaleQtyForGroup` | int | 1 | 组单筛选的最小销量 | 计划文档定义 |
| `replenishRemindDays` | int | 21 | 补货提醒查询天数 | 计划文档定义 |

## 四、字段命名变更说明

| 原字段名（旧BO） | 新字段名（对齐后） | 变更原因 |
|---|---|---|
| `openDayOfWeek` | `publishDayOfWeek` | 文档命名，表意更清晰 |
| `applyBeginTime` | `applyStartTime` | 文档命名，与applyEndTime对称 |
| `coverageWinThreshold` | `directAwardCoverageRate` | 文档命名，表意更精准 |

## 五、小数格式说明

配置JSON中的小数类字段存储为 **百分比的小数形式**，代码使用时需要乘以100转为百分数进行比较：

| 配置字段 | JSON值 | 代码中百分数值 |
|---|---|---|
| `sameSourceWeightRate` | `0.10` | `10.00` (10%) |
| `directAwardCoverageRate` | `0.80` | `80.00` (80%) |
| `recycleCoverageThreshold` | `0.80` | `80` (80%) |

代码示例：
```java
// 小数转百分数
BigDecimal weight = config.getSameSourceWeightRate().multiply(BigDecimal.valueOf(100));
// weight = 10.00 (即10%)
```

## 七、配置读取架构

```
CommonDictConfig (数据库)
    ↑ 查询: business_type='qcListPage', dict_type='consignmentRecruit', key='recruitConfig'
    │
ConsignmentRecruitConfigService (缓存服务, 1分钟TTL)
    ├── getConfig() → ConsignmentRecruitConfigBO
    └── refreshConfig() → 强制刷新缓存
            │
            ▼ 注入到各 Service
    ┌────────────────┬────────────────┬─────────────────┬──────────────────┐
    │CoverageCalc    │AutoAward       │CeTimeoutClean   │RecycleCheck      │
    │  .sameSource   │  .directAward  │  .ceCreate      │  .recycle        │
    │  WeightRate    │  CoverageRate  │  TimeoutDays    │  Threshold       │
    │                │  .awardWaitDays│                  │  .consecutive    │
    └────────────────┴────────────────┴─────────────────┴──────────────────┘
```

## 八、Job Service 与配置项的映射关系

| `ConsignmentRecruitRecycleCheckService.java` | `RECYCLE_COVERAGE_THRESHOLD = 80` | `recycleCoverageThreshold` (0.80 * 100) |
